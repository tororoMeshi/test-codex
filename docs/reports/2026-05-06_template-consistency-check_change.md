# 変更レポート（要件/設計見直し）

## 概要
- 発生日: 2026-05-06
- 対象機能/モジュール: テンプレート構成定義、生成後プロジェクト検査
- 影響範囲: `docs/template-structure.md`, `docs/template-checklist.md`, `scripts/check-template.sh`
- 関係者: AI reviewer / maintainer

## 1. 何が想定外で起きたのか
- `docs/project-intent.md` が必須ファイルとして扱われている一方で、テンプレート構成ドキュメントの最小構成に含まれていなかった。
- 生成後プロジェクトではテンプレート由来の変更レポートを削除する方針だが、自動検査ではテンプレート由来レポート名を明示的に検出していなかった。

## 2. 想定外が起きた原因
- 必須ファイル一覧、チェックリスト、構成ドキュメントの更新が完全には同期していなかった。
- `docs/reports/` は生成後プロジェクトでも利用するため、全ての Markdown を禁止できず、テンプレート由来ファイルだけを識別する規則が必要だった。

## 3. 対応方法
- `docs/template-structure.md` の最小構成に `docs/project-intent.md` を追加した。
- 生成後プロジェクト検査で `docs/reports/template-*.md` と `docs/reports/*_template-*.md` をテンプレート由来レポートとして検出するようにした。
- チェックリストの表現を、自動検査の命名規則と揃えた。

## 4. 要件定義や設計で修正した部分
- テンプレートの必須構成に `docs/project-intent.md` を明示した。
- 生成後プロジェクトでは、将来作成される本物の変更レポートを許容しつつ、テンプレート由来レポートだけを削除対象にする方針へ整理した。
