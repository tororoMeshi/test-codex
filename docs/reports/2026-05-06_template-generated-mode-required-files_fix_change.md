# 変更レポート（要件/設計見直し）

## 概要
- 発生日: 2026-05-06
- 対象機能/モジュール: テンプレート検査
- 影響範囲: `scripts/check-template.sh`, `docs/template-checklist.md`, `docs/template-structure.md`
- 関係者: AIレビュー

## 1. 何が想定外で起きたのか
- `generated` モードで削除を求めるテンプレート本体専用ファイルが、共通必須ファイルにも含まれていた。
- 生成後チェックの禁則語検索が Markdown に限定され、設定ファイルなどの置換漏れを検出できなかった。

## 2. 想定外が起きた原因
- テンプレート本体と生成後プロジェクトで必要なファイルの違いを、検査スクリプト上で分離していなかった。
- 置換漏れチェックの対象範囲が、README の説明する「プロジェクト名など」の差し替え範囲より狭かった。

## 3. 対応方法
- 必須ファイルを共通必須とテンプレート本体専用必須に分けた。
- 生成後プロジェクトではテンプレート本体専用ファイルを forbidden として扱うようにした。
- 生成後の禁則語チェックを Markdown 限定から、除外ディレクトリと主要バイナリ拡張子を除くファイル全体へ広げた。

## 4. 要件定義や設計で修正した部分
- `docs/template-checklist.md` に、テンプレート本体専用ファイルを生成後プロジェクトへ残さない方針を明記した。
- `docs/template-structure.md` の最小構成がテンプレート本体向けであることを明記した。

## 補足（任意）
- 検証方法: `bash scripts/check-template.sh template` と、生成後相当の一時コピーで `bash scripts/check-template.sh generated` を実行する。
