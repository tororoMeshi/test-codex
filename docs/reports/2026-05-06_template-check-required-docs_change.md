# 変更レポート（要件/設計見直し）

## 概要
- 発生日: 2026-05-06
- 対象機能/モジュール: テンプレート健全性チェック、テンプレート構成定義
- 影響範囲: `scripts/check-template.sh`, `docs/template-structure.md`
- 関係者: テンプレート利用者、生成後プロジェクトの保守者

## 1. 何が想定外で起きたのか
- README から参照される `docs/template-checklist.md` と `docs/template-structure.md` が、テンプレート本体の必須ファイルチェックに含まれていなかった。
- 生成後プロジェクトの置換漏れチェックが、追跡済み Markdown のみを対象にしていた。

## 2. 想定外が起きた原因
- テンプレート利用者の入口になる文書と、自動チェック上の必須ファイル定義が同期していなかった。
- CI上のチェックを主に想定していたため、ローカル実行時の未追跡 Markdown 検出が不足していた。

## 3. 対応方法
- `docs/template-checklist.md` と `docs/template-structure.md` を `required_files` に追加した。
- `docs/template-structure.md` でも両ファイルを最小構成の必須ファイルとして扱うようにした。
- 生成後プロジェクトの Markdown 置換漏れチェックを `find + grep` に変更し、未追跡 Markdown も対象にした。
- 空文字パターンが混入しても全ファイル一致扱いにならないよう、空文字はスキップする防御を追加した。

## 4. 要件定義や設計で修正した部分
- README から参照されるテンプレート本体向け文書は、推奨ではなく必須構成として扱う方針に揃えた。
- 生成後チェックは、CIだけでなくローカルの置換漏れ確認にも使える検査として扱う方針に寄せた。
