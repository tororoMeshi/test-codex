# 変更レポート（要件/設計見直し）

## 概要
- 発生日: 2026-05-06
- 対象機能/モジュール: `scripts/smoke-test-generated.sh`
- 影響範囲: 生成後プロジェクト検査のスモークテスト
- 関係者: AI コード生成/修正担当者

## 1. 何が想定外で起きたのか
- 生成後プロジェクトのスモークテストが、作業ツリー全体をコピーして検証していた。
- 失敗ケースでは、`generated` チェックが失敗したことだけを確認しており、期待した禁止文言で失敗したかを確認していなかった。

## 2. 想定外が起きた原因
- ローカル実行時の未追跡ファイルや一時ファイルが、コピー先の生成後プロジェクトに混入しうる構成だった。
- ネガティブケースの検証が終了コードのみで、失敗理由の検証が不足していた。

## 3. 対応方法
- `git archive HEAD` を使い、コミット済みファイルだけから生成後プロジェクト相当の検証ディレクトリを作成するようにした。
- 失敗ケースで終了コード、期待するエラーメッセージ、対象の禁止文言を確認するようにした。

## 4. 要件定義や設計で修正した部分
- テンプレート本体のスモークテストは、ローカル作業ツリーではなく `HEAD` の内容を基準に検証する方針へ明確化した。
- 生成後チェックの失敗ケースは、期待した理由で失敗したことまで検証する方針へ補強した。

## 補足（任意）
- 再発防止策: スモークテストのネガティブケースでは、終了コードだけでなく出力内容も検証する。
- 判断の前提/制約: 未コミット変更は `git archive HEAD` に含まれないため、コミット前の作業ツリー全体の検証用途には使わない。
